When to use Canvas

  • raster graphics
  • games
  • fractals
  • real-time charts
  • image editing
  • pixel-by-pixel manipulation
  • high-performance

When to use SVG

  • structured data
  • user interfaces
  • interactive visualizations
  • lots of text
  • static or server-generated content

How to Make Your SVG

  • by hand, using a text editor
  • by hand, using a vector graphics editor
  • on the server, using one of many libraries
  • in the browser, using a data-oriented framework
    (e.g. Protovis, InfoVis Toolkit, d3)
  • in the browser, using an SVG drawing library
    (e.g. Raphaël, jQuery SVG)

Let's take a look at Raphaël

Raphaël is Awesome

It does not fully implement everything you can do in SVG, but it's great for building interactive vizualizations and UIs.

It nicely encapsulates the raw SVG generation. This way you don't have to worry about XML namespaces and browser-specific JavaScript.

It provides a several convenience methods for doing things beyond just putting graphics on the screen.

Create an SVG Container

Before you can do anything with Raphaël, you have to create your SVG container. All drawing is done with a reference to this object.

Text
